## Added

- **`zurm-server`** — standalone PTY session daemon (`cmd/zurm-server`). Sessions persist independently of the zurm GUI window; attach from any number of clients simultaneously. Communicates over a Unix socket using a length-prefixed binary protocol (`zserver` package).
- **`[server]` config section** — two new fields:
  - `server.enabled` (bool, default `false`) — when true, zurm delegates PTY creation to a running `zurm-server` instance instead of spawning shells directly.
  - `server.address` (string, default `""`) — Unix socket path of the zurm-server. Empty resolves to `~/.config/zurm/server.sock`.
- **Session restore for Mode B** — `session.json` now persists `server_session_id` per pane. On restore, zurm re-attaches to the live server session instead of spawning a new shell, preserving terminal history and running processes across GUI restarts.
- **`make build-server`** — Makefile target that builds the `zurm-server` binary.

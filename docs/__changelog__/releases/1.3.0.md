## Added

- **`zurm-server`** — standalone PTY session daemon (`cmd/zurm-server`). Sessions persist independently of the zurm GUI window; attach from any number of clients simultaneously. Communicates over a Unix socket using a length-prefixed binary protocol (`zserver` package).
- **Per-pane opt-in server mode** — server-backed panes are created on demand via `Cmd+Shift+B` (new server tab). Regular tabs (`Cmd+T`) always use a local PTY. No global `enabled` flag.
- **Auto-start** — zurm-server is spawned automatically when a server-backed pane is opened and the server is not already running. The binary is located next to the `zurm` executable or via PATH. Server output is logged to `~/.config/zurm/server.log`.
- **`[server]` config section** — two fields:
  - `server.address` (string, default `""`) — Unix socket path. Empty resolves to `~/.config/zurm/server.sock`.
  - `server.binary` (string, default `""`) — path to the `zurm-server` executable. Empty = search sibling directory then PATH.
- **Session restore for Mode B** — `session.json` persists `server_session_id` per pane. On restore, zurm re-attaches to the live server session instead of spawning a new shell, preserving terminal history and running processes across GUI restarts.
- **`make build-server`** — Makefile target that builds the `zurm-server` binary.

## Changed

- `server.enabled` config field removed. The server is always available on demand; no opt-in flag is needed.
- `pane.New()` reverts to its original 5-parameter signature (no `serverSessionID`). Server-backed panes use the new `pane.NewServer()` function exclusively.
